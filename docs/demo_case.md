# Demo Case：种草拔草多样例结构迁移 MVP

## 1. Case Overview

本 demo 的目标是基于多条 2-5 分钟种草 / 拔草类样例视频，抽取可迁移的内容结构，再迁移到新的商品或内容主题中。系统关注的是视频的创作结构，例如强 Hook、背景铺垫、问题解释、选购标准、多产品拆解、适合人群、避坑提醒和 CTA，而不是复刻样例视频的具体画面、字幕或表达。

- case_id: `case_01`
- case_name: `种草拔草多样例结构迁移 MVP`
- vertical: `seeding_de_seeding`
- category_mode: `multi_category`
- primary_demo_category: `pet_food`
- source_sample_duration_range: `2-5 分钟`
- target_output_duration: 第一版建议 `60-90 秒`，后续可压缩成 `20-30 秒`短 demo

当前参考视频多为中长种草 / 拔草内容，因此样例结构拆解会使用“相对位置 + 内容槽位”的方式描述。后续系统可以把中长样例结构压缩迁移成更短的视频方案。

## 2. Vertical Scope

当前垂类是“种草拔草向，多种类迁移”。第一版 MVP 选择宠物猫粮作为主 demo，但结构设计需要支持跨品类迁移。

支持迁移的品类包括：

- 宠物用品 / 猫粮
- 服饰穿搭
- 美妆护肤
- 数码产品
- 食品饮料
- 生活用品
- AI 工具 / 软件产品

当前 MVP 不做：

- 不训练模型
- 不复刻原视频具体画面和文案
- 不做完整电商系统
- 不做真实成交转化预测
- 不强依赖真人视频生成

## 3. Reference Samples

当前已有参考视频的角色如下：

**视频 1：服饰搭配教学型种草**

作用：参考“穿搭痛点 → 原因分析 → 单品展示 → 试穿对比 → 搭配方法 → 总结建议”的结构。它适合补充“问题解释、试穿对比、搭配修正”的通用槽位。

**视频 2：大衣选购 / 单品推荐型种草**

作用：参考“购买场景 → 选购标准 → 多款推荐 → 适合人群 → 收藏款分享 → 轻 CTA”的结构。它适合补充“选购标准、多款推荐、适合人群”的通用槽位。

**视频 3：宠物安全警示 / 避坑型内容**

作用：参考“强情绪 Hook → 亲身经历 → 风险解释 → 正确行动建议 → 转发型 CTA”的结构。它适合作为拔草 / 避坑 / 风险提醒结构的主参考。

**视频 4：宠物食品多产品推荐型内容**

作用：参考“品类背景 → 多产品拆解 → 每款卖点 → 适合对象 → 评论互动”的结构。它适合作为多产品种草 / 推荐结构的主参考。

第一版主 demo 选择“宠物用品 / 猫粮”方向，因为视频 3 和视频 4 都在宠物场景，一个偏避坑，一个偏推荐，组合后适合展示“种草 + 拔草”的结构迁移。

## 4. Source Video Structure Assumption

当前样例视频多为 2-5 分钟，不适合用固定的 0-3s、3-6s 短视频结构拆解。本节使用“百分比 + 大致内容段落”的方式描述样例结构，后续真实视频时长变化时仍然可以复用。

### 4.1 宠物安全警示 / 避坑型结构

适合参考视频 3。

1. 0%-10%：强情绪 Hook / 事故结果前置
   - 作用：用高风险事件抓住注意力
   - 示例：猫咪误食线头，差点出大问题
   - 可迁移槽位：`risk_hook`

2. 10%-35%：亲身经历叙述
   - 作用：建立真实感和代入感
   - 示例：发现异常、错误处理、意识到情况不对
   - 可迁移槽位：`story_context`

3. 35%-60%：风险升级与处理过程
   - 作用：展示问题严重性和解决路径
   - 示例：带猫去医院、检查、确认没有针头
   - 可迁移槽位：`risk_escalation`

4. 60%-85%：专业解释 / 避坑知识
   - 作用：告诉用户为什么不能这样做，以及应该怎么做
   - 示例：看到线头不能拽，可能导致肠穿孔
   - 可迁移槽位：`de_seeding_warning`

5. 85%-100%：行动建议 / 转发 CTA
   - 作用：让用户采取行动或分享给目标人群
   - 示例：收好针线牙签，转发给养猫朋友
   - 可迁移槽位：`cta`

### 4.2 宠物食品多产品推荐型结构

适合参考视频 4。

1. 0%-8%：品类背景 / 信任建立
   - 作用：说明为什么有资格推荐
   - 示例：养了五只猫，试过很多品牌，全程无广告
   - 可迁移槽位：`authority_context`

2. 8%-75%：多产品分段拆解
   - 作用：逐个讲产品特点、卖点和适合对象
   - 示例：鲜肉粮、肠胃养护粮、单一肉源粮、烘焙粮
   - 可迁移槽位：`multi_product_breakdown`

3. 75%-90%：选购标准总结
   - 作用：让用户知道不同需求该怎么选
   - 示例：挑食、肠胃敏感、容易过敏、黑下巴等
   - 可迁移槽位：`selection_criteria`

4. 90%-100%：评论互动 CTA
   - 作用：引导用户留言自己的使用体验
   - 示例：你们还喂过哪些好粮，评论区分享
   - 可迁移槽位：`comment_cta`

### 4.3 服饰穿搭教学型结构

适合参考视频 1 和视频 2 作为 secondary reference。

常见结构：

1. 开场：场景和痛点
2. 单品 / 品类介绍
3. 选购标准
4. 试穿 / 使用效果展示
5. 错误示范或常见顾虑
6. 修正方法
7. 多套方案
8. 总结建议

这类结构后续可迁移到：

- 美妆：妆前问题 → 产品试用 → 妆后效果
- 数码：使用痛点 → 产品功能 → 测试结果
- 生活用品：生活问题 → 产品使用 → 前后对比

## 5. Chosen MVP Demo Direction

第一版 demo 方向：

- 主题：新手养猫怎么选猫粮：不同猫咪需求对应不同猫粮，不要盲买
- 视频类型：种草 + 拔草 + 多产品推荐
- 目标用户：新手铲屎官、准备换粮的人、猫咪肠胃敏感或挑食的用户
- 目标时长：第一版结构迁移输出建议 `60-90 秒`；后续为了比赛展示，可以压缩成 `20-30 秒` demo

目标输出：

- 样例结构蓝图
- 新视频脚本
- 分镜
- `timeline_plan`
- 素材缺口报告
- 包装建议

## 6. Expected Transfer Structure

以下是一版适合 60-90 秒结果的迁移结构。它不是最终写死模板，而是第一版 MVP 的默认参考结构。

1. 0-8s：风险 / 痛点 Hook
   - 示例文案：“新手买猫粮，最怕不是买贵了，而是买错了。”
   - 作用：制造问题意识，吸引新手铲屎官继续看

2. 8-18s：问题背景
   - 内容：不同猫咪有不同需求，比如挑食、肠胃敏感、容易过敏、黑下巴、需要长肉
   - 作用：说明不能盲目跟风热门推荐

3. 18-50s：分类推荐
   - 肠胃敏感：看肠胃养护型
   - 容易过敏：看单一肉源
   - 挑食：看适口性和冻干包裹
   - 担心油脂 / 黑下巴：看烘焙粮或低油方案
   - 作用：完成主要种草内容

4. 50-65s：选购标准总结
   - 内容：看肉源、消化率、油脂感、是否适合自家猫体质
   - 作用：从单品推荐上升为方法论

5. 65-78s：避坑提醒
   - 内容：不要只看网红推荐；不要一上来囤大袋；换粮要循序渐进
   - 作用：加入拔草和风险控制

6. 78-90s：CTA
   - 示例文案：“你家猫是什么情况？评论区告诉我，我帮你按需求拆。”
   - 作用：引导评论、收藏或转发

## 7. User Materials

demo 假设下用户已有素材：

- 2-4 张猫粮包装图
- 1 段猫咪吃粮视频
- 1 张成分表截图
- 3-5 条产品卖点文案
- 1 张猫咪日常照片

故意设置的素材缺口：

- 缺少开头强风险画面
- 缺少多产品横向对比图
- 缺少猫咪肠胃不适 / 挑食表现画面
- 缺少结尾 CTA 专用画面

这些缺口用于验证 P0 的素材缺口识别和补全策略能力。

## 8. Expected Gaps and Fill Strategies

1. 缺少开头强风险画面
   - 策略：用猫粮包装特写 + 大字标题 + 快速 zoom + 警示色包装补全

2. 缺少多产品横向对比图
   - 策略：用现有产品图生成对比卡片

3. 缺少猫咪状态画面
   - 策略：用字幕解释 + 图标标签 + 现有猫咪视频裁切补全

4. 缺少 CTA 专用画面
   - 策略：用产品摆拍 + 底部评论引导条补全

## 9. Replacement Plan

本 demo 是第一版可参考 case，不是最终写死内容。后续如果队友提供更好的真实样例视频，只需要替换：

- `source_sample.reference_videos`
- `source_sample.structure_assumption`
- `target_content.topic`
- `target_content.category`
- `user_materials`

不影响后续：

- schema 设计
- API 结构
- mock pipeline
- `structure_blueprint` / `timeline_plan` 的整体思路

真实样例视频到位后，优先把它们放在 `examples/case_01/assets/sample/` 或在 `demo_case.json` 中记录外部链接，再重新跑视频解析和结构拆解。
